SDL Boilerplate

One goal for this class is to create a boilerplate project to function as a starting point for 2D game projects that build to Windows, OS X, iOS and Android. The boilerplate will have the following characteristics.

  • Uses SDL2, SDL2_image and SDL2_ttf. (done)
  • Written in C/C++ and Lua. (done)
  • Generates application installers for OS X, Windows, iOS and Android.
  • Illustrates sprite-based animation using SDL2 textures.
  • Illustrates mouse click and touch event handling.
  • Builds in Xcode and Visual Studio 2010 Express.
  • Provides a built-in console overlay that can be toggled on and off, to allow input of Lua chunks.
  • Includes setup instructions for OS X, Windows and Linux, including instructions on how the library binaries are built.

Extended Boilerplate

The following features can be added to the base boilerplate.

  • Generates installer for Linux.
  • Replace Visual Studio support with Code Blocks or other IDE.
  • Illustrates side scroller character movement.
  • Illustrates the use of Box2D.
  • Provides a very simple GUI library (label, button, text input).
  • Illustrates screen transitions.
  • Provides a compressed atlas for assets (image data, Lua scripts).
  • Illustrates playback of sound effects (memory resident).
  • Illustrates playback of music (stream from file).
  • Add multiplayer support. (Mark Swoope)

OS X Application Bundle

Develop a process that builds the game to an application bundle that can be dragged into the Applications folder for installation without requiring the end user to have SDL installed. See README-macosx.txt in the source distribution of SDL.
