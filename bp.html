Boilerplate Project

Overview

In this activity, you will select a task that needs to get done for the a boilerplate project named sdl-lua2 and start to work on that.

References

  • the sdl-lua2 project

Set Up

In addition to maintaining your course repository, you will also maintain a repository on Github. This Github repository will be a fork of the sdl-lua2 repository. You will then be able to push your work on the project to Github so I can merge your additions.

If you don't already have a Github account, create one now. The free tier service is sufficient for this course.

After logging into Github, fork the sdl-lua2 repository. Then clone your forked repository into your local development environment. Make sure that you can build and run the code.

Start a Task

Write up a short proposal on how you want to contribute to the sdl-lua2 project. Email this to me for feedback and final approval. See the project wiki for ideas on how to contribute to the project.

Add a file named BP.md to your course repository. In this file, place a link to your Github repository and include the text to you approved boilerplate task proposal. That way, I have a record of how to get to your files and a description of what you are working on.
