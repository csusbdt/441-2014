SDL Boilerplate Project

Overview

In this activity, you will select a task that needs to get done for the SDL Boilerplate project and start to work on that.

References

  • SDL Boilerplate Project

Set Up

In addition to maintaining your course repository, you will also maintain a repository on Github. This Github repository will be a fork of the SDL Boilerplate repository. You will then be able to push your work on the Boilerplate to Github so that the Boilerplate maintainers can merge your additions.

If you don't already have a Github account, create one now. The free tier service is sufficient for this course.

After logging into Github, fork the SDL Boilerplate repository. Then clone your forked repository into your local development environment. Make sure that you can build and run the code.

Start a Task

Write up a short proposal on how you want to contribute to the Boilerplate project. Email this to me for feedback and final approval. See the course TODO list for ideas on how to contribute to the project.

Add a file named BP.md to your course repository. In this file, place a link to your Github repository and include the text to you approved boilerplate task proposal. That way, I have a record of how to get to your files and a description of what you are working on.
